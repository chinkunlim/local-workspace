# Open Claw System Fixes & Architecture Updates

I've successfully implemented your requests, repairing the core pipeline bugs and enhancing the inbox manager.

## 1. Orchestrator Caching Bug Resolved
The issue where `audio-transcriber` V8.0 abruptly stopped after Phase 2 without proceeding to Phase 3 has been fixed.
- **Root Cause**: The orchestrator instantiated all Phase classes at startup, meaning Phase 2 (and onwards) cached the initial disk state (where files were still processing) and didn't realize Phase 1 had just completed them.
- **Resolution**: `run_all.py` was refactored to dynamically instantiate each phase immediately before execution. The pipeline now correctly sees the latest state and flows seamlessly through all phases.

## 2. Architecture Streamlining (Phase Cleanup)
We've permanently resolved the functional overlap between the monolithic pipelines and the newer dedicated skills.
- **Audio Transcriber**: Removed Phase 4 (Highlight) and Phase 5 (Synthesis) entirely. It is now strictly a 3-phase pipeline (Transcription, Proofread, Merge).
- **Doc Parser**: Removed Phase 2 (Highlight) and Phase 3 (Synthesis). It now correctly concludes after Phase 1d (VLM Vision Parsing).
- The `state_manager.py` DAG trackers and status dashboards were updated to reflect this modern 3-phase and 4-phase configuration, respectively.

## 3. Inbox Daemon Archiving (Manual Scan)
The daemon no longer misses files that were placed in the `raw` directory while it was turned off.
- **Implementation**: We added a robust `scan_all()` method that retroactively processes any unprocessed files in `data/raw/`. 
- **Auto-Scan on Startup**: The daemon now automatically triggers this full scan every time it starts up, immediately routing any backlog.
- **Manual CLI Support**: Added a `--scan-only` flag to allow you to trigger a manual sorting run without leaving the background service running. 
- *Verification*: Executed `python3 core/inbox_daemon.py --scan-only` successfully. All 35 stranded `.pdf` and `.m4a` files in `data/raw/生理心理學` were safely routed to `audio-transcriber` and `doc-parser` based on the configured regex patterns!
